# GitHub 2500-Repository AGENTS.md Study

GitHub's writeup is useful for practical file shape. The strongest sections to include are commands with flags, testing instructions, project structure, code style with examples, git workflow, and explicit boundaries.

The study also supports the three-tier boundary pattern: always do, ask first, never do. This pattern is easy for agents to apply and easy for humans to review.

Use this reference for template composition. Use the arXiv paper for the stricter warning about unnecessary context and cost.

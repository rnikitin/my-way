# Skill References

This directory holds local reference notes for upstream guidance that should remain canonical outside this repository. Use it for copyrighted articles, product docs, papers, catalogs, and public specs where a short distillation is better than copying the full source.

## Structure

Each file should include the source URL, why the source matters to `my-way`, when to open the upstream source, what to apply locally, and what not to over-copy. These are references, not runnable skills, and they should never be blind link stubs.

## How To Navigate

Use these files when a principle, archetype, or template cites a source and you need the reasoning behind the local recommendation. For runnable workflows, go back to [../_vendored/](../_vendored/) or [../mine/](../mine/). For applied project setup, start from [../../archetypes/](../../archetypes/).

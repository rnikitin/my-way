# Evaluating AGENTS.md Paper

This paper is the cautionary source for instruction minimalism. It reports that extra repository context can increase exploration and cost, and can reduce task success when instructions add requirements that are not useful for the task.

The practical rule is not "always add an AGENTS.md." The rule is: add a short, hand-curated instruction file only when it captures non-derivable constraints that prevent real mistakes.

This reference drives [`../../principles/instruction-minimalism.md`](../../principles/instruction-minimalism.md) and the hard ceiling on archetype `AGENTS.md` templates.

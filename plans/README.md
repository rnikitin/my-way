# Plans

Use this directory for durable implementation plans when work on `my-way` becomes large enough that a future agent needs context beyond a short issue or commit message.

## Structure

Plan files should use date-first names:

```text
YYYY-MM-DD-short-description.md
```

Each plan should include context, assumptions, intended changes, acceptance criteria, ordered steps, validation, decisions, and progress.

## How To Navigate

For current repository status, start with [../HANDOFF.md](../HANDOFF.md). For reusable long-running plan structure, use [../templates/execplan/EXECPLAN.md](../templates/execplan/EXECPLAN.md). Do not put product evidence or source references here when they belong in `skills/_refs/`, `principles/`, or an archetype.
